Abstract

DevOps Pipeline is an academic implementation of a continuous integration and continuous delivery workflow, built to demonstrate how modern software teams move code from a developer's machine into a running, monitored production system without manual bottlenecks.

The project traces a single change through seven disciplined stages — planning, version‑controlled coding, automated building, testing, release packaging, deployment, and live monitoring — mirroring the practice used across the software industry today.

It was undertaken as a collaborative submission for the Computer Science & Engineering (Data Science) programme, combining principles of software engineering, systems automation, and applied data monitoring.
